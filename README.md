# Go Course

Учебный репозиторий с практическими заданиями по Go. Условия — в `lesson1/lesson1.md`, `lesson2/lesson2.md` и `lesson3/lesson3.md`, решения — в соответствующих папках.

**Требования:** Go 1.22+

## Структура

### Lesson 1 — основы

| Папка | Тема |
|-------|------|
| `base/` | Базовый синтаксис, указатели, структуры |
| `maps/` | Map: CRUD, частотный анализ, фильтрация |
| `oop/` | ООП в Go: композиция, инкапсуляция, полиморфизм |
| `sliceTask1/` | Слайсы: общий массив, `len`/`cap`, `append` |
| `sliceTask2/` | Передача слайсов в функции, указатели на элементы |
| `sliceTask3/` | Удаление элементов из слайса |
| `sliceTask4/` | Работа со слайсами (продвинутые задачи) |
| `sliceTask5/` | Слайсы + unit-тесты |

### Lesson 2 — продвинутые темы

| Папка | Тема |
|-------|------|
| `defer/` | `defer`, порядок выполнения, именованные возвраты |
| `errors/` | Ошибки: `errors.New`, `fmt.Errorf`, `%w`, `errors.Is` |
| `generics/` | Обобщённый стек `Stack[T]` |
| `interface/` | Интерфейсы |
| `panics/` | `panic` и `recover` |
| `analysis/` | Разбор поведения кода |

### Lesson 3 — конкурентность и синхронизация

Условия: `lesson3/lesson3.md`

| Папка | Задачи | Тема |
|-------|--------|------|
| `channels/` | task1–task8 | Data race, pipeline, merge, fan-out, tee, декоратор |
| `concurrency/` | task1–task2 | Потокобезопасный кеш, параллельная загрузка данных |
| `syncCond/` | task1–task3 | Очередь с лимитом, ресторан, пул подключений |

| `syncWait/` | task1–task3 | WaitGroup: горутины 0..99, HTTP, business logic |

## Как запускать

Каждая задача — отдельный `package main`. Из папки с файлом:

```bash
go run task1.go
# или
go run .
```

Тесты (если есть):

```bash
go test ./lesson1/sliceTask5/...
go test ./lesson1/oop/task1/...
```

## Полезно помнить

- Слайс передаётся **копией заголовка**, но массив часто **общий** — смотри комментарии в `sliceTask1` и `sliceTask2`.
- `append` меняет `len` — результат нужно **присваивать** обратно.
- `recover` работает только внутри `defer`.
- **Lesson 3:** замыкание в цикле — передавай индекс аргументом: `go func(id int) { ... }(i)`.
- **Lesson 3:** `for range` по каналу зависнет, если отправитель не закроет канал.
- **Lesson 3:** `WaitGroup` — `Add` до `go`, `Done` в `defer`, `Wait` в main.
- **Lesson 3:** из `sync.Pool` наружу отдавай копию (`string(buf)`), не сам слайс из пула.
